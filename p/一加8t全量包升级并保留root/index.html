一加8T全量包升级并保留ROOT

ColorOS 并不好用

May 22, 2021

前言

在试用了一天多 ColorOS 之后, 鉴于其后台管理过于严格, 我又趁午休时间刷回了 OOS, 之前已经有了用 TWRP 保留 root 的全量包更新经验和文章, 本篇只想快速记录一下.

准备工作

  1. 9008 线刷包 (OOS)
  2. payload_dumper
  3. Magisk-v23.0.apk

步骤

步骤一: 刷回 OOS

  1. 进入 开发者选项, 勾选 允许 OEM 解锁, 开启 USB 调试, 然后关机静置.
  2. 同时按住 音量+ 和 音量-, 然后用 USB 线连接电脑, 开启 9008 线刷, 刷机完毕后重启进入系统.

步骤二: 解锁 OOS 并 ROOT

  1. 进入 开发者选项, 勾选 允许 OEM 解锁, 开启 USB 调试, 然后输入以下指令:

    adb devices
adb reboot bootloader
fastboot devices
fastboot oem unlock

    使用 “音量键” 和 “电源键” 确定解锁手机, 然后完成初始化进入系统.

  2. root步骤参考上一篇文章, 此处省略.

步骤三: 更新系统

  1. 下载要更新的全量包, 复制多一份至电脑或手机 (取决你从哪一端下载)

  2. 在电脑端, 使用 payload_dumper 提取 OOS.zip 的 boot.img, 并将其复制到手机存储中.

  3. 安装并打开 Magisk, 选择 “安装”, “选择并修补一个文件”, 选择置入手机的 “boot.img”, 最后点 “开始”, Magisk 会生成一个类似于 magisk_patched-xxx.img 的文件, 将其复制一份到电脑:

  4. 安装系统更新, 安装完毕后重启进入系统桌面:

    检查 root 权限, 发现此时是没有 root 的:

  5. 刷入 magisk_patched-xxx.img, 根据以下指令完成:

    adb devices
adb reboot bootloader
fastboot devices
fastboot flash boot magisk_patched-xxx.img

    刷入完成后, 重启进入系统, 打开 “设置” 以及 “Magisk Manager” 查看系统信息以及 root 情况:

结尾

至此, 成功尝试了 OOS 的全量包更新并保持 ROOT 权限, 之前是依靠 TWRP 以及 Magisk 中的模块自动化完成的, 这次全部自己操作, 还是积累到一点经验的.
